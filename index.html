O problema que você está enfrentando é comum em aplicações web que utilizam a Web Audio API. O som "para de funcionar" geralmente porque os navegadores (especialmente no Android/iOS) suspendem o AudioContext para economizar bateria ou processamento quando ele fica ocioso por muito tempo, ou por questões de segurança. No seu código, você está criando um novo AudioContext toda vez que a função tocarBip é chamada. Isso é pesado e faz com que o navegador eventualmente bloqueie a criação de novos contextos. Como resolver: Crie um contexto global único: Não crie um novo contexto a cada bip. Retome o contexto (Resume): Sempre verifique se o contexto está "suspenso" antes de tocar. Feche os osciladores: Garanta que os nós de áudio sejam desconectados para liberar memória. Aqui está o código corrigido para a sua seção de áudio: JavaScript // --- SISTEMA DE ÁUDIO OTIMIZADO --- let audioCtx = null; function obterAudioContext() { if (!audioCtx) { audioCtx = new (window.AudioContext || window.webkitAudioContext)(); } // Se o navegador suspendeu o áudio por inatividade, retomamos if (audioCtx.state === 'suspended') { audioCtx.resume(); } return audioCtx; } function tocarBip(frequencia, duracao, volume = 0.1) { try { const ctx = obterAudioContext(); const oscillator = ctx.createOscillator(); const gainNode = ctx.createGain(); oscillator.type = 'sine'; oscillator.frequency.setValueAtTime(frequencia, ctx.currentTime); // Envelope suave para evitar estalos e liberar memória gainNode.gain.setValueAtTime(volume, ctx.currentTime); gainNode.gain.exponentialRampToValueAtTime(0.0001, ctx.currentTime + duracao); oscillator.connect(gainNode); gainNode.connect(ctx.destination); oscillator.start(); oscillator.stop(ctx.currentTime + duracao); // Limpeza de memória após o som acabar oscillator.onended = () => { oscillator.disconnect(); gainNode.disconnect(); }; } catch (e) { console.error("Erro ao tocar som", e); } } const somClique = () => tocarBip(880, 0.1, 0.1); const somTrocaReferencia = () => tocarBip(440, 0.3, 0.15); Por que isso acontece? Limite de Instâncias: Os navegadores permitem apenas um número limitado de AudioContexts (geralmente entre 6 e 50). Como você criava um novo a cada clique, o limite era atingido rapidamente em um rali longo. Políticas de Autogestão: Dispositivos móveis são agressivos com abas em segundo plano ou apps que ficam abertos muito tempo sem interação direta com o DOM (como é o caso de um GPS de rali que fica apenas visualizando). Dica Extra para Rali: Como o celular pode entrar em modo de economia de energia, certifique-se de que o usuário clicou em algo (como o botão "Iniciar Prova") para ativar o áudio. O navegador proíbe sons automáticos sem um primeiro clique humano. Seu código já faz isso no btnIniciar.onclick, o que é ótimo. Substitua o bloco // --- SISTEMA DE ÁUDIO --- antigo por este novo e o som deverá permanecer estável durante toda a prova. podemos deixar mais agudo? o anterior estava melhor de ouvir Com certeza! Para deixar o som mais agudo, precisamos aumentar o valor da frequência (o primeiro parâmetro da função). No código anterior, o bip de clique estava em 880Hz e o de troca em 440Hz. Para um som mais "cortante" e fácil de ouvir em ambientes barulhentos (como dentro de um carro de rali), podemos subir para a casa dos 1500Hz a 2500Hz. Aqui está o ajuste com frequências mais altas e um tempo de ataque levemente mais nítido: JavaScript // --- SISTEMA DE ÁUDIO MAIS AGUDO E NÍTIDO --- let audioCtx = null; function obterAudioContext() { if (!audioCtx) { audioCtx = new (window.AudioContext || window.webkitAudioContext)(); } if (audioCtx.state === 'suspended') { audioCtx.resume(); } return audioCtx; } function tocarBip(frequencia, duracao, volume = 0.1) { try { const ctx = obterAudioContext(); const oscillator = ctx.createOscillator(); const gainNode = ctx.createGain(); // 'square' ou 'triangle' são mais fáceis de ouvir que 'sine' em ruído oscillator.type = 'sine'; oscillator.frequency.setValueAtTime(frequencia, ctx.currentTime); gainNode.gain.setValueAtTime(volume, ctx.currentTime); // Finaliza o som de forma limpa gainNode.gain.exponentialRampToValueAtTime(0.01, ctx.currentTime + duracao); oscillator.connect(gainNode); gainNode.connect(ctx.destination); oscillator.start(); oscillator.stop(ctx.currentTime + duracao); oscillator.onended = () => { oscillator.disconnect(); gainNode.disconnect(); }; } catch (e) { console.error(e); } } // AJUSTE DE TONS: // 2000Hz é bem agudo e "fura" o barulho do motor const somClique = () => tocarBip(2000, 0.08, 0.1); // 1200Hz para a troca (um pouco mais grave que o clique, mas ainda agudo) const somTrocaReferencia = () => tocarBip(1200, 0.25, 0.15); O que mudou: Frequência do Clique: Subiu de 880 para 2000 (Som de "bi" bem agudo). Frequência da Troca: Subiu de 440 para 1200 (Ainda é um alerta claro, mas distinto do clique). Duração: Aumentei levemente o tempo do clique (0.08) para que o ouvido perceba melhor o impacto. Dica técnica: Se o som ainda estiver "baixo", você pode trocar o oscillator.type = 'sine'; para oscillator.type = 'triangle';. A onda triangular tem mais harmônicos e é muito mais audível em dispositivos móveis sem precisar aumentar o volume real do aparelho.
